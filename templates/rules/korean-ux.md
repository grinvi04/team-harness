# 한국어 UI/UX 규칙 (한국어 화면)

> 한국 사용자 대상 화면 문구는 영어 직역체·서구 기본값을 쓰지 않는다. 단일 출처: `docs/korean-ux.md`.
> (영어 UI 서비스에는 미적용.)

## 마이크로카피 — 해요체
- 버튼: 동사 단독 ("저장" "삭제" "확인" "취소" "닫기" "검색"). **영어 버튼명(Save·Cancel·Delete) 금지.**
- 토스트: ~했어요 ("저장했어요" "삭제했어요"). 에러: ~이에요/~해주세요 ("필수 항목이에요").
- 확인 다이얼로그: ~할까요?/~하시겠어요? ("정말 삭제하시겠어요? 되돌릴 수 없어요.").
- 합쇼체(~하시겠습니까?)는 약관·공식 공지에만.

## 용어 — 직역 금지 (정착어 사용)
- 마이페이지(○) / 내 페이지·프로필 페이지(✕)
- 장바구니(○) / 쇼핑 카트(✕)
- 문의하기(○) / 연락하기(✕)
- 회원가입(○) / 가입·사인업(✕)
- 비밀번호(○) / 패스워드(✕)·아이디(○) / 유저네임(✕)
- 이용약관(○) / 서비스 약관(✕)·개인정보처리방침(○) / 프라이버시 정책(✕)
- 공지사항(○) / 뉴스·알림(공지 의미)(✕)
- (정착 외래어 허용: 로그인·로그아웃·대시보드·필터·알림)

## 포맷
- 날짜: 화면 표시 `YYYY.MM.DD`(점) — API 전송은 ISO 8601 유지.
- 원화: `1,234,567원` / `₩1,234,567` (달러 기호·소수점 금지).
- 휴대폰: `010-XXXX-XXXX` (하이픈 자동). 사업자번호 `XXX-XX-XXXXX`.
- 이름: 단일 필드(성+이름 분리 없음). 주소: 우편번호 검색→도로명+상세 2필드.
- 다이얼로그 버튼: 취소(왼쪽) → 확인/삭제(오른쪽). 금액은 우측 정렬.
